# AgentGuard 工程交付说明

## 运行前提

所有 Python 命令使用工作区根目录的虚拟环境：

```powershell
E:\study\code\agent_guard\venv\Scripts\python.exe
```

如果该环境不存在，先在工作区根目录创建并安装依赖：

```powershell
cd E:\study\code\agent_guard
python -m venv venv
.\venv\Scripts\python.exe -m pip install -r .\agentguard\backend\requirements.txt
```

不要把 API Key、token、证书或真实密钥写入代码、manifest、测试样本或 Docker 镜像。运行配置通过环境变量注入。

`demo_data/benign/.env` 是固定演示占位文件，只允许 `demo-secret-not-real` 和本地 SQLite 示例值。`scripts/scan-secrets.ps1` 会检查高置信密钥格式、非白名单 `.env`、常见凭证文件和可疑 secret 赋值。

## 交付检查

从仓库根目录 `E:\study\code\agent_guard\agentguard` 执行：

```powershell
.\scripts\verify.ps1
```

只检查后端：

```powershell
.\scripts\verify.ps1 -SkipFrontend
```

Docker 检查：

```powershell
.\scripts\verify-docker.ps1
```

单独执行密钥扫描：

```powershell
.\scripts\scan-secrets.ps1
```

单独验证提交文件集合：

```powershell
.\scripts\verify-release-files.ps1
```

单独验证当前代码生成的 benchmark 指标：

```powershell
.\scripts\verify-benchmark.ps1
```

单独验证正式结果 artifact：

```powershell
.\scripts\verify-results.ps1
```

需要实际启动容器并做健康检查：

```powershell
.\scripts\verify-docker.ps1 -RunContainers
```

最终审计：

```powershell
.\scripts\final-audit.ps1
```

如果当前机器没有启动 Docker Desktop，可先执行本地审计：

```powershell
.\scripts\final-audit.ps1 -SkipDocker
```

## 结果归档

正式、可写入报告的实验结果必须单独放到 `results/<实验名称>_<时间戳>/`。临时过程记录和下一步计划放到 `tmp-plans/`。

Benchmark API 会返回运行环境、git commit、样本数、消融结果和压力测试摘要。正式归档时保留原始 JSON，不只截图前端表格。

正式结果写入报告前必须通过：

```powershell
.\scripts\verify-results.ps1
```

推荐使用脚本归档：

```powershell
.\scripts\save-benchmark.ps1 -ExperimentName "main_benchmark"
```

## 部署说明

本地开发：

```powershell
.\scripts\start-backend.ps1
.\scripts\start-frontend.ps1
```

一键启动和停止本地演示：

```powershell
.\scripts\start-demo.ps1
.\scripts\start-demo.ps1 -Wait
.\scripts\check-demo.ps1
.\scripts\stop-demo.ps1
```

标准库演示服务：

```powershell
.\scripts\start-backend.ps1 -DemoServer
```

Docker：

```powershell
docker compose up --build
```

容器后端使用非 root 用户运行，审计数据库写到 `/data/agentguard_audit.db`。前端容器通过 nginx `/api` 反向代理访问后端。

CI 工作流位于 `.github/workflows/ci.yml`，覆盖本地最终审计、Docker compose config 和 Docker build。

提交前拆分审查和验收边界见 `docs/提交与验收清单.md`。

## 可选安全开关

`AgentGuardGateway` 支持 `enforce_consistency_precheck=True`。开启后，工具执行前会进行 manifest + 静态入口函数一致性检查，高风险或严重偏差会被拒绝执行。默认 API 不开启该开关，正式实验通过 `benchmark.json` 中的 `consistency_enforcement` 字段报告门控效果。该开关是保守静态门控，不替代运行时证据采集。
