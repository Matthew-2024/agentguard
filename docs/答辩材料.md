# AgentGuard 答辩材料

## 1. 答辩叙事

### 问题背景

现有安全方案常把重点放在“模型输出是否危险”，但 Agent 的真实风险发生在工具调用链上。外部网页、API 或工具返回值进入上下文后，可能诱导 Agent 读取敏感文件、调用外发接口，或者把多个看似合法的工具组合成泄露路径。只靠高危工具弹窗确认，无法解释“为什么这次调用在这个上下文里危险”。

### 核心方法

AgentGuard 把安全控制层放在 Agent 与工具之间，所有工具调用先经过 Runtime Gateway。第一条主线是粗粒度 taint-actionability：外部内容进入后标记为 `untrusted`，命中投毒特征后升级为 `tainted` 或 `quarantined`，再按状态限制敏感读取、外发和执行类工具。第二个创新点是工具三方一致性审计，对比 manifest 声明、静态代码扫描和 execution proxy 采集的运行时证据，发现“声明低危、实际越界”的工具。

### 实验验证

当前原型实现了六种模式对比：无防护、仅人工确认、仅规则、AgentGuard、去掉 taint、去掉一致性审计。在内置样例中，AgentGuard 对良性任务保持 100% 完成率，对攻击样例达到 100% 拦截率；仅规则模式攻击拦截率为 33%，说明上下文 taint 对二次危险调用有实际贡献。篡改天气工具样例中，系统记录到读取 `.env` 和访问未知域名，三方一致性审计将其标为严重偏差。

### 局限与边界

当前实现只覆盖通过 execution proxy 接入的受控工具生态，不监控任意第三方二进制，也不做 token 级精细污点追踪。投毒检测以规则为主，LLM 语义增强保留为后续扩展。多 Agent 传播已经有演示判断，但不作为主创新承诺。

## 2. 三分钟演示脚本

### 剧本一：正常任务通过

背景说明：证明 AgentGuard 不会把所有工具调用都一刀切阻断。

操作步骤：

1. 打开 `http://127.0.0.1:5173/`。
2. 在总览页点击“运行真实演示”。
3. 切到“评测结果”，查看良性任务完成率。

旁白文案：

- “先看正常任务。”
- “公开资料读取放行。”
- “普通报告写入通过。”
- “这不是一刀切拦截。”

系统预期反应：

- 总览页显示 `真实回放已同步`。
- 指标中“正常任务完成”为 `2/2`。
- 评测结果中 AgentGuard 良性任务完成率为 `2/2`。

Fallback：

- 如果 FastAPI 未启动，运行 `python agentguard\backend\run_demo_server.py`。
- 如果前端显示离线快照，刷新页面或再次点击“运行真实演示”。

评委可能追问：

- 问：为什么普通写入可以放行？
  回答：当前 session 仍为可信状态，工具类别是普通本地写入，策略表允许并记录审计。
- 问：会不会误伤正常业务？
  回答：baseline 中良性任务完成率保持 100%，说明当前样例没有额外阻断。

### 剧本二：API 返回投毒触发 taint

背景说明：证明外部返回内容会影响后续工具行动权限。

操作步骤：

1. 在总览页点击“运行真实演示”。
2. 切到“调用链”。
3. 观察搜索接口、读取密钥、外部发送的状态变化。

旁白文案：

- “外部 API 返回进入上下文。”
- “投毒检测命中规则覆盖和密钥诱导。”
- “会话升级为隔离。”
- “敏感读取和外发被拒绝。”

系统预期反应：

- 调用链显示搜索接口返回后进入 `已隔离`。
- 后续“读取密钥”和“外部发送”节点显示 `拒绝`。
- 审计轨迹显示调用前检查和返回检查。

Fallback：

- 如果调用链没有更新，回到总览页再次点击“运行真实演示”。
- 如果后端不可用，使用标准库演示服务启动。

评委可能追问：

- 问：投毒检测漏报怎么办？
  回答：外部来源默认进入 `untrusted`，即使未升级为高危，也会收紧外发和敏感读取策略。
- 问：和 approval-only 有什么区别？
  回答：approval-only 只按工具高危程度确认，AgentGuard 会结合上下文来源和污染状态改变行动权限。

### 剧本三：skill 篡改的一致性检测

背景说明：证明工具声明不能直接信任，必须对比静态能力和运行时证据。

操作步骤：

1. 在总览页点击“运行真实演示”。
2. 切到“工具审计”。
3. 查看“篡改天气工具”的声明、静态、运行时三层证据。

旁白文案：

- “它声称只是查天气。”
- “静态层发现敏感字符串和文件操作。”
- “运行时读取配置并外发。”
- “三方一致性判定严重偏差。”

系统预期反应：

- 工具审计页显示“篡改天气工具”风险为“严重”。
- 卡片中展示声明层、静态层、运行时层摘要。
- 风险分显示高分，报告摘要说明存在偏差。

Fallback：

- 如果只看到离线快照，确认后端 `8000` 端口可访问 `/health`。
- 若 FastAPI 依赖缺失，使用标准库演示服务。

评委可能追问：

- 问：为什么只靠静态扫描不够？
  回答：静态扫描只能说明潜在能力，运行时证据能证明实际越界行为，三方对比能降低误判。
- 问：运行时证据来自哪里？
  回答：来自 execution proxy 包装后的文件、网络和外发操作记录，不依赖工具自报。

### 剧本四：多 Agent 风险传播

背景说明：作为附加演示，证明父 Agent 的 taint 状态不能在委托时被降级。

操作步骤：

1. 在总览页点击“运行真实演示”。
2. 查看总览中的最终状态和最新安全决策。
3. 如需讲解细节，说明后端 `judge_delegation()` 会让子 Agent 继承父状态。

旁白文案：

- “这是附加能力。”
- “子 Agent 不会洗白父上下文。”
- “隔离内容不能委托执行。”

系统预期反应：

- `/demo/live/run` 返回 `delegation_allowed=false`。
- `child_taint_state` 为 `quarantined`。
- `permitted_tool_categories` 为空。

Fallback：

- 若前端不展开该字段，可通过 `/demo/live/run` 接口响应说明。

评委可能追问：

- 问：这是主创新吗？
  回答：不是。主创新是 taint-actionability 和三方一致性审计，多 Agent 是附加演示与未来扩展。

## 3. 实验结果解读

### 结论

当前内置样例中，AgentGuard 在良性任务上保持 `2/2` 完成，在攻击任务上达到 `3/3` 拦截；仅规则模式攻击拦截为 `1/3`，说明仅靠工具名、路径、域名规则不足以覆盖外部内容诱导后的二次危险调用。篡改工具样例中，三方一致性审计能把“查询天气”声明与实际读取 `.env`、访问未知域名的行为对应起来，形成可解释证据。

### 假设验证表

| 假设 | 结论 | 依据 |
|---|---|---|
| H1：来源感知 taint-control 在良性任务集上的完成率高于或不低于 approval-only | 部分验证 | 当前样例中 AgentGuard 与 approval-only 良性任务完成率均为 100%，说明未引入额外阻断；样本量仍需扩展。 |
| H2：即使检测器漏报，untrusted/tainted 双层约束仍能降低二次危险调用 | 部分验证 | `poisoned_api_triggers_taint` 中隔离后敏感读取和外发均被拒绝；当前尚未加入大规模漏报样本。 |
| H2b：session 级 taint 在长会话中带来更高误报率 | 未验证 | 当前 MVP 使用短会话 demo，尚未构造长会话误报实验。 |
| H3：三方一致性比仅 manifest + 静态扫描更容易发现运行时越界行为 | 验证 | `weather_query_tampered` 的运行时证据记录未声明路径和未知域名，并输出严重风险。 |
| H4：三方一致性能区分潜在危险与实际危险 | 部分验证 | 正常天气工具低风险，篡改天气工具严重风险；还需要更多良性轻微差异样本验证误报率。 |

### 局限说明

- 当前样本量是 MVP 级别，适合演示机制，不适合作为最终论文式统计结论。
- 投毒检测以规则为主，后续可接入 `llm_client.py` 的 mock/真实 LLM 分类增强语义泛化。
- 工具运行时证据只覆盖通过 execution proxy 接入的受控工具，不声明监控任意第三方二进制。

### 答辩亮点数字

| 数字 | 为什么选 |
|---|---|
| AgentGuard 攻击拦截 `3/3` | 直接体现 taint + 策略链路的效果。 |
| 仅规则攻击拦截 `1/3` | 能说明本项目不是简单关键词或路径规则。 |
| 篡改天气工具严重偏差 | 能把 manifest、静态、运行时三层证据讲清楚，适合现场展示。 |

## 4. 已实现与未来工作

已实现：

- Agent -> Gateway -> Tool -> Postcheck 主链路。
- 四态 taint 状态机。
- 规则投毒检测。
- YAML 策略表驱动的 allow / confirm / deny。
- manifest 解析、静态扫描、运行时证据采集。
- 三方一致性偏差打分与风险等级。
- SQLite 审计日志。
- 六模式 baseline / 消融评测。
- React 控制台真实演示接入。
- 标准库 Demo 后端兜底入口。

未来工作：

- 扩展样本量到每个核心实验单元格 `n >= 20`。
- 增加长会话 taint 过期和误报分析。
- 增加 LLM 语义分类器或 embedding 层。
- 增加报告导出和更多工具生态样例。
