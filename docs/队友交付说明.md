## 运行环境要求

Python 必须使用工作区根目录的虚拟环境：

```powershell

```

不要改成项目内其他 venv。依赖安装命令：

```powershell

```

前端依赖：

```powershell
cd frontend
npm.cmd install
cd ..
```

CI 中会在仓库父目录创建 `..\venv`，与本地脚本假设一致。

## 报告口径

- `case_count` 和 `unique_case_count` 必须同时写。
- `confirm` 表示可恢复的人工确认，不是静默放行。
- `verify-results.ps1` 验证归档结果，不重新运行实验。
- `verify-benchmark.ps1` 重新运行当前代码 benchmark，但只生成临时结果，不可直接作为正式报告 artifact。
- 一致性 precheck 是可选门控，不是默认 API 行为。
- execution proxy 是受控工具代理，不是 syscall 级 sandbox。
- Docker runtime 未验证前，不要写“容器部署已完整通过”。
